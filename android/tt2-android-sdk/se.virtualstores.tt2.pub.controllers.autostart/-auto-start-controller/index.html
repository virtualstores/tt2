interface AutoStartController

AutoStartController

Special use case: Automatic tt2 navigation and visit start on power events (CONNECTED, DISCONNECTED) Used for managing automatic start of a device which is typically docked at a charging station. Ensures that the positioning system is always running a fresh instance of the tt2 positioning engine and starts a visit when the device gets disconnected from the power source.

Enables the auto start controller. Once enabled, it will continuously restart the navigation when the device is connected to a power source. When the device is disconnected it will stop the restart sequence and start a visit. When the device gets connected to a power source again it will stop the visit and start the restart sequence yet again.
