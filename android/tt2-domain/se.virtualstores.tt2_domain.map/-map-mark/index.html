Parameters

id

Needs to be unique.

position

The location of the mark.

floorLevelId

(Optional) Which floorLevel the mark should render on.

data

(Optional) Generic data type where you can put any data that you want to access.

deletable

Danger: Creates a permanent mark. App needs to be restarted to remove it.

triggerRadius

(Optional) if set the MarkerController will trigger events when user enters/exits the radius of this mark.

clusterable

Determines if the mark should be clustered if it overlaps with another mark, i.e if you zoom out the map to that point.

visible

if the mark is visible or not.

focused

if the mark is focused or not. Focused marks gets scaled up to a bigger size.
